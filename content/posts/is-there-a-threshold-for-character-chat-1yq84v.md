# Character Chat有门槛吗?

> AI character chat类的产品到目前为止没有任何核心资产

# 核心资产

什么是核心资产? 核心资产指代的是价值大, 难迁移, 难复制的产品功能, 技术实现, 使用惯性. 某种意义上来说指代的是价值/当前模式下的不可替代性.

举个例子: 微信的核心资产是什么? 是庞大的用户量吗? 是顶级的技术能力? 是垃圾的产品力"审美"? 我认为微信的核心资产有两个:

1. 基于即时聊天/内容分享的社交习惯
2. 基于支付/收款/转账的经济能力

在我上大学的时候, 我实际上是使用QQ更多, 但是为什么现在更多的使用微信呢? 因为工作之后发现, 周围的人都在用微信. 因为工作之后发现, 一些转账/红包相关的事情, 只能在微信上操作.

我在这里提出一个想法: **普适性的产品, 用户量的上限取决于对于用户层级的包容性**

你可以批评微信做的很烂, 但是必须要承认一点就是, 哪怕是一个从没有使用过智能机的人, 也可以很快速的使用微信的绝大部分功能. 微信能够容忍几乎全部的用户层级.

当初微信宣传能够免费打电话/发短信真是一步妙手.

# 行业分析

> 人类始终没有意识到, 自己有多么喜欢戏剧.

当ChatGPT问世后, 最容易被利用的一种方式: Character对话, 即LLM扮演某个角色, 来和用户进行对话.

后面随着LLM的越狱能力 以及 开源模型的喷涌, 两种类型的Character出现了:

1. NSFW: 色情/暴力/政治
2. SFW: 影视/幻想 角色

### 产品介绍

在现在的阶段, 一些代表性的产品:

1. character.ai 老牌的ai产品, 有目前最丰富的character资源, 多模态体验良好
2. [janitor](https://janitorai.com/) 丰富的character资源, 不错的多模态体验, 支持自用ai proxy接入
3. crushon 比较丰富的character资源, 仅文本对话
4. yodayo 基本同上, 但是有对二次元特攻

### 创作者生态

我认为目前这些产品是缺少远见来构建创作者生态的, 一个可能的原因是, 创作内容非常容易被复制, 通过一些提示词破解的方式, 能够轻易的获取character的设计以及产品的系统提示词, 这对于创作者和平台来说来说是毁灭性的打击: 内容可以被轻易的破解, 甚至是轻易的复制.

LLM是比较特殊的: 系统提示词当然重要, 但是context message同样重要. 所以类似crushon.ai的memory应该也被认为是一种创作.

现在的创作者的实际体验: 创建character -> 发布character -> 结束. 或者是 聊天 -> 分享聊天 -> 结束.创作逻辑时不闭环的.

创作者从中获得了什么? 成就感 以及 ? Nothing

现在的创作体验是不好的, 因为很难像其他的内容社区一样从中获利. 没有广告, 没有商单, 甚至一些平台是不允许出现广告的. 从长远计, 这其实是会导致高质量的产出者越来越少.

丰厚的回报可能性吸引高质量的创作者.

除非将来用AI生产社区内容, 但是目前看来, AI生产的内容质量比较一般.

### 技术难度分析

现在的ai陪伴不存在真正的技术门槛, 说到底都只是调用提供商的API, 谁比谁强呢?

误以为是技术门槛的门槛:

1. NSFW, [其实很简单](https://github.com/verazuo/jailbreak_llms)
2. RolePlay, 参考开源的[SillyTavern](https://github.com/SillyTavern/SillyTavern)

随着底层模型的进步, Prompt的质量高低影响会越来越低, 而真正影响体验的是Charcter自身的精彩程度

# 发展方向

### 情节

小说是靠情节推进的, 在现在的ai陪伴中, 是不存在情节的, 一切就是一个固定的背景, 发生的一切在百十句对话之后就变得无聊.

这里给大家推荐一个非常有趣的产品: [HaiJoy](https://haijoy.com/) HaiJoy中有一个`Dream`模式, 我觉得非常好玩.

简单了来说就是你可以假装和Character经历一些事件, 这些事件实际上会影响你们的后续对话.

我觉得这是一个非常有意思的设计(直到离职前, 我都认为这是一个非常有趣的设计)

### 多模态

GalGame, RPG 这些典型的游戏模式, 告诉了我们一点经验, 处理好的剧情设计之外

1. 美术设计
2. 音乐设计
3. 配音

是非常影响用户沉浸度以及体验的

对应到现在的技术能力下, 对应的是:

1. 文生图
2. BGM
3. TTS

这些能力是可以实现的, 但是成本会很比较高

这里还是要说一句, [HaiJoy](https://haijoy.com/)是支持这些能力的, 但是很可惜, 没有人在乎.

在SillyTavern中, 实际上支持live2D, 以及3D的角色引入, 我认为这会是一个趋势, 尽管现在的行业标准还没有确定下来

# 总结

1. AI陪伴没有技术壁垒, 起步早, 用户量大的产品, 大概率能够继续领先地位
2. LLM目前最适合的场景就是RolePlay
3. Charcter的创作是易于复制/抄袭的, character的数量不会是一个有效的标准
4. LLM底座实际上决定了这些产品的上限, 而不是Prompt/Memory

# 闲谈

### 小冰

小冰是天才, 小冰 -> 虚拟男女友 -> 小冰岛 -> 克隆人, 我认为产品眼光是超过任何从业者

其中的一些产品设计非常值得思考, 笔者在小冰工作的两年感受是, 起步很早, 概念很多, 但是底层技术确实落后了

我认为小冰需要一个强力的算法基底, 在此基础上才有可能实现自己的产品目标

如果低下骄傲的头颅, 我认为小冰可以创造一个更伟大的概念
